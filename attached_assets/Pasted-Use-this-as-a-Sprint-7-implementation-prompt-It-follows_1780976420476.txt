# SPRINT 7 — PRODUCTS & SERVICES MANAGEMENT

## IMPORTANT EXECUTION RULES

Do NOT start coding immediately.

Follow this process exactly:

### Phase 1 — Audit

Read existing code and produce a validation report.

Do NOT modify files.

Verify:

* Existing products routes
* Existing services routes
* Database tables
* OpenAPI schemas
* Generated React Query hooks
* DashboardProvider context
* Dashboard routes
* Existing business ownership checks
* Existing upload/image infrastructure
* Existing pagination patterns
* Existing search integration

Produce a report showing:

* What already exists
* What is missing
* Potential blockers
* Recommended implementation order

Wait until audit is complete before writing code.

## Sprint Goal

Business owners must be able to:

### Products

* Create products
* Edit products
* Delete products
* View product list
* Mark products active/inactive
* Add product price
* Add description
* Add images
* Manage inventory status

### Services

* Create services
* Edit services
* Delete services
* View service list
* Set price ranges
* Add descriptions
* Add images
* Mark active/inactive

All functionality must be restricted to business owners.

# Phase 3 — Ownership Security

Verify every mutation is protected.

Products:

Only business owners may:

* create
* edit
* delete

Services:

Only business owners may:

* create
* edit
* delete

If ownership middleware does not exist:

Create reusable middleware.

Example:

requireBusinessOwner

This middleware should:

* verify authenticated user
* verify ownership in businessOwners table
* reject unauthorized access

Use one shared implementation.
